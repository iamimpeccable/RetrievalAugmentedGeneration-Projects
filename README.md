<h1>Project-1:</h1>
<h3> Built a Retrieval-Augmented Generation (RAG) pipeline from scratch, and it is called Chat with a PDF. <br>
     The problem RAG solves: LLMs like Claude/GPT don't have access to your documents. RAG fixes this by: </h3>
    <ol>Splitting your PDF into small chunks </ol>
    <ol>Embedding each chunk (converting text → numbers that capture meaning)</ol>
    <ol>Storing those embeddings in a vector database </ol>
    <ol>When you ask a question → finding the most relevant chunks → sending them to an LLM as context </ol>
    <ol>The LLM answers based on your document — not just its training data</ol>

<h3>Think of it like giving the LLM a "cheat sheet" pulled from your PDF before it answers.</h3>
